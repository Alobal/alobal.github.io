点云序列深度学习

发表于 2022-01-15 分类于 深度学习 阅读次数： Valine：

点云数据格式

目前已有的大部分视觉识别工作都是在二维的基础上，而二维图像数据，本质上对环境鲁棒性不强，很容易受到光照等因素的影响。相对来说，三维数据则更好的描述了物体的几何结构信息，而且对环境影响更加鲁棒。

点云是三维传感器的原始数据，同时表示格式简单。如果选择其他三维描述格式可能会损失很多精度，例如图像只能表达一个视角的3D数据。对于Mesh，可能要选择面片类型，面片大小，连接方式。对于网格，要选择网格大小，大了会损失精度。

传统的点云特征都是人对于特定任务进行手工构造的，对于一个新的任务很难去继续使用这样的特征，因此进入到了深度学习时代，希望通过深度学习自动学习特征。

点云是一种不规则的数据，因此有几种转换点云的方法来深度学习。

栅格化：的方式来表示点云，然后可以使用3D CNN[1]来抽取特征。然而有一些很大的缺点：

  • 3D CNN 计算量非常大，空间时间复杂度随着分辨率都是三次方增长。因此分辨率提不了多高，一般30x30x30。
  • 量化噪声错误，本质上限制了识别精确度。

投影法：将三维数据投影到二维平面，然后用2DCNN处理[11]，然而如何选择投影的方式，角度，都不是一个简单的问题。同时投影本质上也会丢失一些三维信息。

手工提取点云特征，再简单的应用全连接：然而这种网络的性能被手工特征的局限给限制住了，而不理想。

近期的趋势则是直接在点云数据格式上进行深度学习。

点云帧内学习

空间邻域聚类

点的局部邻域作为一个小整体，可以表达出一些细节上的结构特征。

邻域密度问题：点云不像图像，图像的邻域都是规则化的，然而点云是离散的，不同区域的点密度完全不一样，而如果采用固定的半径构造邻域则会出现密度问题。PointNet++[3]提出了Multi-scale grouping（MSG)，Multi-resolution grouping（MRG）两种方法来对自适应密度构造邻域。

抽取邻域特征： PointNet缺失对局部邻域特征的学习，PointNet++[3]则加强了这一部分，其在局部点云上使用PointNet来抽取特征，即生成了一个可以代表这个局部的特征点，然后逐级往上进行层级式抽取特征。

空间维度编码方法

Permutation不变性：静态点云可以看作是一个无序集合。作为集合类型的无序数据，它不能像图像那种规则数据一样，顺序地进行卷积操作，因为点云不同的输入顺序会在传统卷积下产生不同的结果。因此，PointNet[2]提出可以在点云上使用一个对称函数操作，即可保持点云的置换不变性。PointNet在实验中表明max pooling是最有效的一种对称函数[2]。

Transform不变性：点云的平移旋转不应该影响点云的特征学习。PointNet[2]增加了一个数据变换模块T-Net，来自动学习如何将点云对齐。另外PointNet也同时将T-Net应用在特征维度上。

点云序列帧间学习

点云序列在时间维度上的学习有两种思路，一种是空间时间串行编码，先对每个帧进行空间编码，再对一系列的帧编码进行序列学习。另一种是空间时间编码并行处理，此时时间编码需要寻找每个点对应的时间邻域，并且在时间邻域上聚合相应的特征信息。

搜索时间邻域

图像，骨架，网格之类的规则数据，有着序列帧间的对应关系，进而可以轻易地找到跨时间的对应关系。由于点云是不规则数据，直观上不存在帧与帧之间点的对应关系，进而如何搜索跨时间的邻域是点云序列学习中一个核心的问题。

一种直觉的思路是直接搜索Direct grouping： PointLSTM[5]使用了最简单的Direct grouping，其直接在不同的帧中基于同一个坐标点搜索k近邻。PSTNet[10]则加强了这一方法，提出 Point Tube 。Point Tube在前后相邻帧中搜索固定半径\(r\)作为邻域，以形成一个 tube形状的邻域结构。MeteorNet[4]提出一种动态搜索半径\(r\)的Direct grouping，主要思想在于 "时间过得越久，点移动的就越远" 这一简单的事实，进而去搜索同一个坐标点半径为\(R=r(|t^\prime - t|)\) 。的邻域。，即搜索半径与跨越的时间帧成正比关系。

第二种思路是 基于轨迹预测：由于物体的运动往往有轨迹趋势，因此PointLSTM[5] 提出的 Aligned grouping 通过反向流估计，如FlowNet[12]来计算上一帧中的点。 MeteorNet[4]提出的Chained-flow grouping 在反向估计的基础上，同时使用真实的过去帧来在一定程度上对更远的估计帧进行正则。然而如何进行反向流估计也是一个难题。

另外还可以将每一帧的点云视为一个整体，而可以跳过寻找单个点的时间邻域的步骤，如PointLSTM-PSS[5]中即对整个点云编码成一个特征和隐状态。

时间维度编码方法

虽然基于时间邻域聚类可以聚合一些短期相邻帧的联系，但是其缺乏捕捉长期时间联系的能力。

考虑到RNN和LSTM在序列模型上的大获成功，因此 PointLSTM[5] 通过在点云序列上建立LSTM层来获取点云序列的长期联系。考虑到时间聚类问题，PointLSTM建立了两种编码模型：简化版本 PointLSTM-PSS ，其跳过了时间聚类问题，将一帧的整个点云视为一个词元进行隐状态的编码，并且进入LSTM层进行更新计算，这种方式简单但是能够取得很不错的效果。完整版本 PointLSTM 对一帧内的每个点视为一个词元，具有单独的隐状态和编码。对任意一个点的隐状态计算需要利用到前一帧中该点邻域的所有点，因此计算量较大，但效果超过PointLSTM 0.5~1%。

另一个序列学习上大获成功的模型是Transformer[8]。Transformer的核心self-attention层是一个顺序无关的操作，，因此很适合进入点云领域。PCT: Point cloud transformer[6] 和 Point Transformer [7] 致力于在点云序列上移植Transformer模型。

PCT提出 Naive PCT，其对Transformer进行了简单的修改应用。 Naive PCT 将整个点云视为一个句子，将每个点视作一个词元，并且使用了point embedding和 Self-attention进行学习。由于 self-Attention中Query，Key，Value的计算本身是顺序无关的，且softmax和加权累积也是顺序无关的，因此整个 self-attention 操作是顺序无关的，可以直接用于点云学习。

为避免点云的仿射变换影响，改进版PCT提出使用 Offset-Attention 来对 self-attention的输出再进一步进行残差编码：\(F_{out}=OA(F_{in})=LBR(F_{in}-F_{sa})+F_{in}\)，并且其评分函数由Scale Softmax改成了softmax + l1_norm 以强化注意力权重。另外，和PointNet类似，Naive PCT 的point embedding无法进行局部结构的学习，因此 PCT还提出了 neighbor embedding 来聚合邻域信息，其核心是在embedding空间进行采样+KNN的聚类，然后将邻域与中心点的相对特征进行拼接输出。

PointTransformer和 PCT 类似，首先是在注意力计算中增加了一个基于位置差分的position encoding \(\mathbf{y}_{i}=\sum_{\mathbf{x}_{j} \in \mathcal{X}(i)} \rho\left(\gamma\left(\varphi\left(\mathbf{x}_{i}\right)-\psi\left(\mathbf{x}_{j}\right)+\delta\right)\right) \odot\left(\alpha\left(\mathbf{x}_{j}\right)+\delta\right)\) 。 另外基于残差网络思想， PointTransformer 提出了 point transformer block来输出每个点的特征向量。

SequentialPointNet[9]提出一种 Temporal position embedding， 通过将Transformer[8]的三角函数位置编码应用到点云的时间序列上，然后再通过共享MLP层直接对帧序列进行编码，来形成一种加强时间信息的时间特征放大模块。最终通过 Hierarchical pyramid maxpooling 进行一种类似于多卷积核的池化操作来抽取序列特征。

PSTNet[10]通过时间空间计算解耦的想法，提出了 Point Spatio-Temporal Convolutional(PST卷积)，即完全独立空间卷积核和时间卷积核，使它们互不依赖：

\[ \boldsymbol{F}_{t}^{\prime(x, y, z)}=\sum_{k=-\lfloor l / 2\rfloor}^{\lfloor l / 2\rfloor} \mathbf{T}_{k} \cdot \sum_{\left\|\left(\delta_{x}, \delta_{y}, \delta_{z}\right)\right\| \leq r} \mathbf{S}^{\left(\delta_{x}, \delta_{y}, \delta_{z}\right)} \cdot \boldsymbol{F}_{t+k}^{\left(x+\delta_{x}, y+\delta_{y}, z+\delta_{z}\right)}, \]

并且由于点云的不规则分布，点云邻域与卷积中心的距离是连续变化的，难以使用传统卷积的权重模块来对这些邻域点进行加权。因此 PSTNet将空间卷积定义为一个关于offset的计算函数 \(f\left(\left(\delta_{x}, \delta_{y}, \delta_{z}\right) ; \boldsymbol{\theta}\right)=\boldsymbol{\theta}_{d} \cdot\left(\delta_{x}, \delta_{y}, \delta_{z}\right)^{T} \cdot 1 \odot \boldsymbol{\theta}_{s}\)，以实现对任意offset点的加权操作。

骨架序列学习

在骨架的时空图卷积的基础上，DDGCN[13]认为骨架的空间层级结构和动作的时间序列结构都包含了顺序信息，然而大多数ST graph都是用了无向图结构，即无视了顺序信息，因此DDGCN提出了有向图骨架结构 Directed Spatial-Temporal Graph (DSTG) 。通过有向图中父子节点的定义，父节点的动作实际上会影响到子节点的动作，因此DDGCN在有向图的基础上定义了 bone features \(f_i^B=\overrightarrow{v_{i-1}v_i}=f_{i-1}-f_i\) 来表示父子节点的影响特征。 另外针对图卷积的邻域不确定性，DDGCN提出 Dynamic Convolutional Sampling (DCS) 来对一个节点的邻居列表进行动态的排序，形成动态邻域关系。然而卷积核的权重是有顺序的，而邻居列表是动态变化的，可能会造成权重分配的错序。因此DDGCN使用了一个 Dynamic Convolutional Weights (DCW) 模块来对邻居列表和权重列表进行一个Dynamic Time Warping (DTW)距离的最小化排序，再根据这个排序进行权重分配。

无论是传统无向图骨架还是有向图骨架，都有着结构上的缺陷。静态的图结构限制了表达动态节点关系的能力，因此 SkeletonTransformer[15]完全抛开了传统的静态骨架图结构，而是使用Self-attention来动态查找节点关系。SkeletonTransformer 提出了 Spatial Self-Attention (SSA)模块 和 Temporal Self-Attention (TSA) 模块。 SSA 模块在一个帧内对所有关节点进行自注意力编码，最终输出一个带有动态节点关系的特征。TSA 模块则是在时间上进行注意力计算，同样去找出动态的帧间关系。最终SkeletonTransformer使用了一个 Two-Stream Spatial–Temporal Transformer (ST-TR) Network通过两个并行的运算流来结合两个模块，以得到一个时间空间上信息都很完整，并且不相互依赖的时空特征。
